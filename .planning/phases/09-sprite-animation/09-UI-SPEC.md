## Animation State Map

| Game state | Condition | Section | Frame count | Loops |
|-----------|-----------|---------|-------------|-------|
| Idle / floor | `samusY >= canvasHeight * GAME.floorRatio - 1` | `idle` | 1 | Static (no animation) |
| Spin jump | airborne AND `animState.isScrewAttack === false` | `spinJumpR` | 9 | Yes — `frame % 9` |
| Screw attack | airborne AND `animState.isScrewAttack === true` | `screwAttackR` | 9 | Yes — `frame % 9` |
| Fallback (PNG not loaded) | `spritesRef.current.samus === null` | N/A — shape render | N/A | See Fallback Shape Spec |

**Screen coverage (D-04):** Real sprite used on idle screen, gameover screen, and playing screen once `spritesRef.current.samus` is loaded. No mixed shape/sprite states.

**Fallback coverage (D-05):** When `spritesRef.current.samus === null`, shape-Samus renders on all screens. No error thrown.

## Timing Contract

### Frame Advance Rate

| Animation | FPS | Frame duration | Basis |
|-----------|-----|----------------|-------|
| Idle | N/A (static, 1 frame) | N/A | Single frame, no accumulator needed |
| Spin jump (`spinJumpR`) | **10 fps** | 100 ms (0.1 s) | Claude's discretion — 8-12 fps typical for Super Metroid spin jump; 10 fps chosen as midpoint |
| Screw attack (`screwAttackR`) | **10 fps** | 100 ms (0.1 s) | Same rate as spin jump; distinct visually via overlay, not speed difference |

### dt Accumulator Rules

```
FRAME_DURATION = 1 / 10  →  0.1 seconds

Per-frame loop (airborne only):
  animState.accumulator += dt
  if accumulator >= FRAME_DURATION:
    accumulator -= FRAME_DURATION      // DO NOT reset to 0 — preserves leftover
    frame = (frame + 1) % section.frames
```

**Why subtraction, not reset:** Resetting to 0 discards leftover time and causes drift at 120 Hz displays. Subtracting preserves remainder — timing stays accurate at both 60 Hz and 120 Hz.

**Accumulate only while airborne.** When on the floor, do not advance the accumulator.

### Transition Rules

| Transition | Action |
|-----------|--------|
| Floor → airborne | `frame = 0, accumulator = 0` — start spin jump from frame 0 |
| Spin jump → screw attack (`pendingScrewAttack` consumed) | `isScrewAttack = true, frame = 0, accumulator = 0` — clean start for screw attack animation |
| Airborne → floor landing | `isScrewAttack = false, frame = 0, accumulator = 0` — full reset |
| Game restart (Effect B re-runs) | AnimState is re-declared in closure → auto-resets without explicit reset logic |

## AnimState Contract

Lives in the **Effect B rAF closure** — not a `useRef`. Auto-resets on game restart per D-10.

```typescript
interface AnimState {
  frame: number;        // 0-indexed frame within current section (0-8 for 9-frame sections)
  accumulator: number;  // seconds elapsed since last frame advance
  isScrewAttack: boolean; // true when player re-jumped while airborne
}

// Initial value:
let animState: AnimState = { frame: 0, accumulator: 0, isScrewAttack: false };
```

### Cross-Closure Bridge (D-03 + Pattern 4 from RESEARCH.md)

`handleInput` is outside Effect B's closure and cannot write to `animState` directly. Bridge via a new field on `GamePhysicsState`:

```
pendingScrewAttack: boolean  →  initialize false in createInitialGameState
```

In `handleInput`: if `game.samusVY !== 0` at jump time, set `game.pendingScrewAttack = true`.
In Effect B loop: if `game.pendingScrewAttack`, consume it — set `animState.isScrewAttack = true`, reset frame and accumulator, clear the flag.

## Pixel Scaling Rules

All rules apply to every `drawImage` call that renders from the sprite sheet.

| Rule | Value | Enforcement |
|------|-------|-------------|
| Interpolation | `ctx.imageSmoothingEnabled = false` | Set immediately **before** each `drawImage` — not once at init |
| Coordinate precision | All 9 `drawImage` arguments through `Math.floor()` | Fractional px causes sub-pixel blur even with smoothing disabled |
| CSS reinforcement | `image-rendering: pixelated` on canvas element | Already set in `SamusRunGame.tsx:341` — belt-and-suspenders |
| Scale source | `GAME.samusScale` multiplier | Already plumbed into shape calls; sprite draw respects same multiplier |

### Source Rect Calculation

```
sx = Math.floor(frameIndex * cellSize + contentOffset)   // = frameIndex * 96 + 17
sy = Math.floor(section.sy + contentOffset)               // = section.sy + 17
sw = contentSize  // 81
sh = contentSize  // 81
```

### Destination Rect Calculation

```
dw = Math.floor(contentSize * scale)   // = Math.floor(81 * GAME.samusScale)
dh = Math.floor(contentSize * scale)   // = Math.floor(81 * GAME.samusScale)
dx = Math.floor(x - dw / 2)           // center-X anchor
dy = Math.floor(y - dh)               // bottom-Y anchor (samusY = bottom of sprite)
```

**Anchor contract:** `samusY` is bottom-of-sprite semantics (floor position). `dy = Math.floor(y - dh)` places the bottom of the drawn sprite at `samusY`. This matches `drawSamusIdle`'s anchor convention.

### DEBUG_HITBOX Workflow (D-07)

```
DEBUG_HITBOX = false   // constant in drawSamusSprite (or drawScene)
                       // flip to true locally to render the tuning overlay
```

**Debug overlay spec:**
- `strokeStyle = "rgba(255, 0, 0, 0.8)"` — red, 80% opacity
- `lineWidth = 1`
- Position: `strokeRect(Math.floor(x - hw/2), Math.floor(y - hh), hw, hh)`

- Tuning goal: rect hugs the visible sprite body, excluding arm cannon and transparent padding
- After tuning: bake final constants into `COLLISION`, set `DEBUG_HITBOX = false`, commit

## Screw Attack Visual Overlay

Overlay is drawn **after** `drawSamusSprite` when `animState.isScrewAttack === true` and airborne.

| Property | Value | Rationale |
|----------|-------|-----------|
| Composite operation | `"screen"` | Additive blend — brightens sprite without covering it |
| Fill color | `rgba(80, 120, 255, 0.35)` | Cool blue tint — consistent with project's cool-tone palette constraint |
| Coverage | Same `dx, dy, dw, dh` bounding box as the sprite draw | Exactly matches sprite cell, not hitbox |
| Fallback | If `"screen"` blend looks wrong visually, use `ctx.globalAlpha + fillRect` at opacity 0.25 | Low-stakes visual; adjust at implementation time |

## drawScene Signature Extension

Current signature (Phase 8): `drawScene(ctx, screen, width, height, physics?)`

Extended signature (Phase 9):

```typescript
function drawScene(
  ctx: CanvasRenderingContext2D,
  screen: GameScreen,
  width: number,
  height: number,
  physics?: GamePhysicsState,
  sprites?: { samus: HTMLCanvasElement | null; bg: HTMLImageElement | null },
  animState?: AnimState
): void
```

| Call site | sprites arg | animState arg |
|-----------|-------------|---------------|
| Effect B (rAF loop) | `spritesRef.current` | `animState` (in-scope closure var) |
| Effect A (idle/gameover static render) | `spritesRef.current` | `undefined` — defaults to frame 0 |

When `animState` is `undefined`, `drawSamusSprite` defaults to idle section, frame 0.

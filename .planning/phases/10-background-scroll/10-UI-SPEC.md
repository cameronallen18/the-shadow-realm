## Scroll Behavior Contract

### BG_SCROLL_SPEED Constant

| Property | Value | Notes |
|----------|-------|-------|
| Constant name | `BG_SCROLL_SPEED` | Added to `constants.ts` |
| Value | **70** CSS px/s | Ambient feel — slow enough to read as atmosphere, fast enough to be clearly moving. Mid-point of D-45 recommended range (60–80 px/s). |
| Independence | NOT derived from `speedMultiplier` | Decoupled per D-05 — game speed changes do NOT affect background speed |
| Reference | `PHYSICS.baseScrollSpeed = 220` CSS px/s | Background is ~32% of obstacle speed — intentionally calm |

### Offset Accumulation

Lives in the **Effect B rAF closure** — same pattern as `animState` per D-06.

```typescript
// Declared inside Effect B, alongside animState
let bgScrollOffset = 0;

// Each frame, before drawScene call:
bgScrollOffset = (bgScrollOffset + BG_SCROLL_SPEED * dt) % TILE_WIDTH;
```

**Why `% TILE_WIDTH`:** Offset is bounded to [0, 512). Avoids floating-point growth over long play sessions.

**Auto-reset on restart:** `bgScrollOffset` is a `let` inside Effect B closure. When the screen transitions to "playing", Effect B re-runs and `bgScrollOffset` starts at 0. No explicit reset logic needed.

## Tile Draw Loop Contract

### Draw Order

1. **Background tiles** (new in Phase 10) — drawn first, covers full canvas
2. **Lava floor** (existing in `drawEnvironment`) — drawn on top of tiles, covering bottom 15%
3. **Ceiling stalactites** (existing in `drawEnvironment`) — drawn on top of tiles, along top edge
4. **Obstacles and Samus** (existing) — drawn last

The sky fill (`NORFAIR.sky`) and midground fill (`NORFAIR.midground`) in `drawEnvironment` are **replaced** by the tile draw. The lava and stalactite calls stay.

### Tile Draw Loop

```typescript
function drawBackground(
  ctx: CanvasRenderingContext2D,
  bg: HTMLImageElement,
  width: number,
  height: number,
  offset: number
): void {
  ctx.save();
  ctx.imageSmoothingEnabled = false;

  const TILE_WIDTH = 512;
  const tileH = Math.floor(height); // scale tile to fill full canvas height
  const startX = -Math.floor(offset);

  // Draw enough tiles to cover full width plus one extra for seamless wrap
  for (let x = startX; x < width; x += TILE_WIDTH) {
    ctx.drawImage(
      bg,
      Math.floor(x), 0,
      TILE_WIDTH, tileH
    );
  }

  ctx.restore();
}
```

**Boundary condition:** Loop starts at `-Math.floor(offset)` (which is in range `[-511, 0]`) and steps by `TILE_WIDTH` until `x >= width`. This always draws the leftmost partial tile plus enough full tiles to reach the right edge. No blank gap possible.

**Seamless wrap:** Because `offset` is bounded to `[0, 512)` via modulo, the leftmost tile always starts at `[-511, 0]` — the partially-visible left edge is always a valid tile slice.

### Pixel-Perfect Rules (consistent with QUAL-01)

| Rule | Value |
|------|-------|
| Interpolation | `ctx.imageSmoothingEnabled = false` inside `ctx.save()/restore()` |
| Coordinate precision | All `drawImage` arguments through `Math.floor()` |
| Stretch | Tile is scaled to full canvas height (`height` px) — background covers the entire canvas |

## drawEnvironment Modification

Current signature (Phase 9): `drawEnvironment(ctx, width, height)`

**Option A (chosen per Claude's Discretion):** Add a `drawBackground` helper called *before* `drawEnvironment`. Modify `drawEnvironment` to skip the sky/midground fills when a background image is provided.

Modified drawEnvironment:

```typescript
export function drawEnvironment(
  ctx: CanvasRenderingContext2D,
  width: number,
  height: number,
  bg?: HTMLImageElement | null,
  bgOffset?: number
): void {
  if (bg) {
    drawBackground(ctx, bg, width, height, bgOffset ?? 0);
  } else {
    // Fallback: existing solid fills
    ctx.fillStyle = NORFAIR.sky;
    ctx.fillRect(0, 0, width, height);
    ctx.fillStyle = NORFAIR.midground;
    ctx.fillRect(0, 0, width, height * 0.3);
  }

  // Lava floor (bottom 15%) — always drawn
  const floorY = height * 0.85;
  ctx.fillStyle = NORFAIR.lavaGlow;
  ctx.fillRect(0, floorY, width, height - floorY);

  // ... rest of lava shimmer, ground line, stalactites unchanged
}
```

This preserves the fallback behavior (D-11) and keeps all existing callers valid — passing no `bg` arg continues to render the solid-fill environment.

```typescript
function drawScene(
```

```typescript
  height: number,
  physics?: GamePhysicsState,
  sprites?: { samus: HTMLImageElement | null; bg: HTMLImageElement | null },
  animState?: AnimState,
  bgScrollOffset?: number   // NEW — undefined or 0 = static background
): void
```

| Call site | bgScrollOffset arg | Result |
|-----------|-------------------|--------|
| Effect B (rAF loop) | `bgScrollOffset` (in-scope closure var) | Scrolling background during play |
| Effect A (idle/gameover static render) | `undefined` (or `0`) | Static background — "world not yet moving" |

When `bgScrollOffset` is `undefined`, treat as `0` (no scroll). All existing call sites remain valid without modification beyond adding the arg.

## Static Screen Behavior (D-09)

| Screen | bgScrollOffset | Result |
|--------|---------------|--------|
| Idle | 0 (no arg) | Background frozen at offset 0 — "world hasn't started" |
| Playing | accumulates each frame | Background scrolls |
| Game over | 0 (no arg) | Background frozen — "world stopped" |

This creates a natural semantic: still world → player starts → world moves → player dies → world stops.

## Fallback Contract (D-11)

When `spritesRef.current.bg === null`:

- `drawEnvironment` receives no `bg` argument
- Falls through to existing solid fills: `NORFAIR.sky` + `NORFAIR.midground`
- Lava, shimmer, ground line, and stalactites render as before
- No error thrown, no visual regression

**Verification:** Game must render identically to Phase 9 when `norfair_upper.png` is missing or fails to load.

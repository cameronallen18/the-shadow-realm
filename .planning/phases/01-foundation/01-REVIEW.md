# Phase 01: Code Review Report

**Reviewed:** 2026-04-13T00:00:00Z
**Depth:** standard
**Files Reviewed:** 16
**Status:** issues_found

## Summary

These are the `create-next-app@next-15-3` scaffold files for a personal landing page targeting Vercel. The configuration is broadly sound — TypeScript strict mode is on, App Router is used, Tailwind v4 is wired up correctly via PostCSS. Three issues need attention before development proceeds: a font override bug that silently defeats Geist, missing `next-themes` integration that makes all `dark:` Tailwind utilities inert, and placeholder metadata that would ship to production as-is.

---

## Warnings

### WR-01: Body font-family override defeats Geist everywhere

**File:** `app/globals.css:25`

**Issue:** The `body {}` rule sets `font-family: Arial, Helvetica, sans-serif` explicitly. This wins over any Tailwind utility that references `--font-geist-sans` (including `font-[family-name:var(--font-geist-sans)]` in `page.tsx` and the `--font-sans` mapping in `@theme inline`). Geist loads correctly via `next/font` and its CSS variables are injected, but the explicit `font-family` declaration on `body` takes cascade precedence, meaning the font never actually renders.

**Fix:** Remove the `font-family` line from the body rule. Geist Sans is already applied via the `--font-sans` theme variable and the className on `<body>` in `layout.tsx`.

```css
body {
  background: var(--background);
  color: var(--foreground);
  /* Remove: font-family: Arial, Helvetica, sans-serif; */
}
```

---

### WR-02: `dark:` Tailwind utilities are inert — `next-themes` not installed

**File:** `app/page.tsx:8,30,45` / `package.json`

**Issue:** Tailwind v4's `dark:` variant activates on the presence of a `.dark` class on `<html>`. Nothing in this codebase sets that class — there is no `next-themes` integration, no `ThemeProvider`, and `layout.tsx` does not apply a dark class. The `@media (prefers-color-scheme: dark)` block in `globals.css` correctly switches CSS custom properties, but the `dark:invert`, `dark:hover:bg-[#ccc]`, `dark:bg-white/[.06]`, etc. classes in `page.tsx` will never activate. Per `CLAUDE.md`, `next-themes` is listed as a required dependency for exactly this reason.

**Fix:** Install and configure `next-themes`. Wrap `layout.tsx` body in a `ThemeProvider` with `defaultTheme="dark"` (or `"system"`) and `attribute="class"`.

```bash
npm install next-themes
```

```tsx
// app/layout.tsx
import { ThemeProvider } from "next-themes";

export default function RootLayout({ children }: Readonly<{ children: React.ReactNode }>) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body className={`${geistSans.variable} ${geistMono.variable} antialiased`}>
        <ThemeProvider attribute="class" defaultTheme="dark" enableSystem>
          {children}
        </ThemeProvider>
      </body>
    </html>
  );
}
```

### WR-03: Scaffold placeholder metadata will ship to production

**File:** `app/layout.tsx:15-18`

**Issue:** `metadata.title` is `"Create Next App"` and `metadata.description` is `"Generated by create next app"`. These are the literal `create-next-app` defaults. This site deploys to Vercel from GitHub on push — any merge to main will serve these strings as the page `<title>` and meta description to search engines and social sharing previews. There is no CI guard to catch this.

**Fix:** Replace with the actual site title and description before the first production deploy.

```tsx
export const metadata: Metadata = {
  title: "the shadow realm",
  description: "Personal landing page and project hub.",
};
```

## Info

### IN-01: `@types/node` pinned to v20, Vercel runs Node 22

**File:** `package.json:18`

**Issue:** `"@types/node": "^20"` will resolve to Node 20 types. Vercel's build environment defaults to Node 22 for new projects as of 2025. This creates a gap where Node 22 global APIs have no TypeScript definitions. It causes no runtime errors but will produce type errors if any Node 22 API is used.

**Fix:** Bump to `"@types/node": "^22"`.

---

### IN-02: Scaffold page content is still live

**File:** `app/page.tsx`

**Issue:** The entire page body is `create-next-app` promotional content (Vercel deploy CTA, Next.js docs links, etc.). This is expected for a scaffold starting point, but it is the current live content at the Vercel URL. It is not a code defect, but it is the first thing to replace when building out the landing page.

**Fix:** Replace `app/page.tsx` content with the actual landing page in the next phase.
